UPDATED VERSION — MVP PLANNING
PURPOSE
This document captures all unresolved ambiguities and functional questions identified for
the MVP.
STRATEGIC RECOMMENDATION
Move Game & Lesson Payments Off-Platform or Implement Escrow
Suggestion: We recommend that the core payment system included in the MVP focuses on
Pro and Coach verification subscriptions. All other payments (such as pro game fees and
coaching fees) can be handled physically outside the app between the participants to speed
up MVP launch.
Reasoning & Challenges with Standard In-App Payments: Because games and sports
occur physically outside the app, processing standard payments in-app introduces significant
scam and dispute risks. For example:
If a Pro player organizes a game and another player participates but later claims they
didn't participate and asks for a refund, the platform has no proof of whether they are
telling the truth.
If a Pro organizer starts a game but does not show up to play, and also fails to cancel the
game in the app, it leaves participants with disputed charges.
Alternative In-App Solution (Escrow Method):
Secure Escrow System: If we decide to keep lessons and pro match payments within
the system, we can set up an Escrow payment method or similar. In this scenario, the
platform securely holds the user's payment until all conditions (like verifying attendance or
match completion) are met, before releasing the funds to the coach or player. This
provides a highly secure, trust-building experience and significantly reduces dispute risks.

New MVP Payment Flow & Workarounds (If going Off-Platform):
The Payment Flow: When a player wants to join a paid game, they will be shown the
external payment methods provided by the Pro organizer. The user pays the match fee
externally and uploads a screenshot of the transaction into the app. Once the organizer
verifies the receipt, the user officially joins the lobby.
Liability Disclaimer: Upon clicking "Join Game," users must accept a disclaimer stating
they are entirely responsible for the transaction and that the platform is not liable for
external payments.
Team vs. Individual Matching: Pro organizers will have the option to set up a game as
"Individual Sign-ups" (standard match fee per person) or a "Full Team Request" (a single
bulk match fee for an opposing team).
Dispute Handling & Fake Receipts: To avoid manual admin overhead, the platform will
rely on a peer-reporting system. If an organizer reports a fake receipt (or a user reports
an organizer for ignoring a valid payment), it registers as a strike. Users/organizers with
multiple flags will face automated permanent bans.
