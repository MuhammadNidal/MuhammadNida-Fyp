Let’s Play — Mobile App Project Scope
1. Product Overview
Let’s Play is a mobile platform that helps people discover, organize, and participate in sports and game-
based activities with other users nearby.
The platform connects users based on the sports, games, locations, schedules, and skill levels they are
interested in. Users can find casual games, create their own games, follow other players, message
individuals or groups, and participate in paid or free lessons hosted by coaches or professionals.
The platform also supports verified coach and pro profiles, allowing qualified users to offer lessons, host
premium games, and build a trusted presence within the community.
Example Use Case
A user wants to play basketball next Tuesday afternoon near Chelsea, Manhattan. They open the app,
search for basketball games near Chelsea, apply filters for date and time, and discover available games or
players who are interested in playing. If no suitable game exists, the user can create one and invite or
attract others to join.
2. Product Goals
The primary goals of Let’s Play are to:
Help users find people to play sports and games with.
Help users discover nearby games based on location, time, sport, and preferences.
Allow users to create and manage their own games.
Enable users to follow and communicate with other players.
Support lessons and premium experiences hosted by verified coaches and pros.
Build trust through account verification, ratings, profiles, and role-based verification.
Create a monetization path through paid games, paid lessons, and recurring subscriptions for coach
and pro status.

4. Role Definitions and Permissions
4.1 Player
A player is the default user type on the platform.
Players can:
Create games.
Search for games.
Join games.
Search for other players.
Follow other users by adding them to their roster.
Message other users directly.
Participate in group messages.
View upcoming, past, and created games.
Leave games they joined.
Rate other users after participating in the same game.
Apply for account verification.
Player verification means the platform has taken reasonable steps to confirm that the user is a real person.
4.2 Coach
A coach is a verified user who is allowed to offer lessons on the platform.
Coaches can:
Do everything a player can do.
Create lessons.
Set lessons as free or paid.
Maintain a coach status through a monthly or yearly subscription.
Coach verification should confirm that the user is qualified to give lessons in the selected sport or game.
Coach verification may require a lighter review process than pro verification.
4.3 Pro
A pro is a verified user with professional-level experience in a sport or game.

Coach Confirms that the user qualifies to give lessons.
Pro Confirms that the user has professional experience in the sport or game.
Verification is manually reviewed by the platform team or admins.

6. Primary User Flows
6.1 Login and Onboarding
User opens the app.
User creates an account or logs in.
User completes onboarding.
User selects sports/games of interest.
User lands on the main app experience.
6.2 Search for a Game
User opens Explore.
User selects Game Explorer.
User searches or browses games.
User applies filters.
User selects a game.
User views game details.
User joins the game if spots are available.
6.3 Join a Free Game
User opens a game detail page.
User reviews the details.
User taps Join.
User is added to the game.
Game appears under Upcoming Matches.
User gains access to game messages.
6.4 Join a Paid Lesson or Paid Pro Game
User opens a paid lesson or pro-status game.
User reviews the details and price.
User taps Join.
User completes payment.
User is added to the activity.
Activity appears under Upcoming Matches.
User gains access to relevant messages/comments.
6.5 Create a Game
User opens Create.
User selects the type of activity they want to create.
User enters game details.
User publishes the game.

8. Suggested MVP Scope
The full product vision includes games, player discovery, messaging, ratings, verification, payments,
lessons, subscriptions, and admin tooling.
For an MVP, the recommended scope should focus on proving the core marketplace behavior: users can
find people to play with and organize games.
MVP Features
Recommended MVP features:

Player profiles.
Game discovery.
Game filters.
Game detail screen.
Join game.
Leave game.
Create game.
Upcoming, past, and created games.
Game messages/comments.

Coach and Pro Participation

Paid Activity Payment Timing

11.2 Remaining Open Product Questions
The following questions should be answered before finalizing detailed product requirements:
Can users remove participants from a game they created?
What detailed permissions should moderators have?
What happens when a paid user leaves or cancels?
What refund policy applies to paid lessons and paid pro games?
What specific information is required for player verification?
What specific information is required for coach verification?
What specific information is required for pro verification?
What are the rating criteria?
Are ratings public, private, or partially visible?
Are coaches and pros verified per sport/game, or globally across the platform?
What fees does the platform take from paid lessons or paid games?
What moderation tools are required for user safety?

12. Out of Scope for Initial MVP
The following features are recommended to be out of scope for the initial MVP unless they are critical to the
business model:
Paid lessons.
Paid pro games.
Coach/pro subscriptions.
Automated verification.
Full admin dashboard.
Ratings.
Group messaging.
Advanced notification settings.
Complex refund flows.
Advanced recommendation algorithms.
Skill-based matchmaking.
13. Success Criteria
The MVP should be considered successful if users can:
Create a profile.
Find relevant games near them.
Create a game.
Join a game.
Communicate with other participants in a game.
View their upcoming, past, and created games.
Discover and follow other players.
The broader product should be considered successful if it can reliably support community-driven sports
discovery, paid lessons, verified coaches and pros, safe user interactions, and repeat engagement.
14. Summary
Let’s Play is a sports and games discovery platform built around the idea of helping people find others to
play with.
The initial product should focus on the core player experience: creating games, discovering games, joining
games, and connecting with other players.
Coach, pro, payment, subscription, verification, and admin workflows are important to the long-term
business model, but they should be carefully phased so the MVP can validate the core marketplace behavior
before adding operational complexity.
